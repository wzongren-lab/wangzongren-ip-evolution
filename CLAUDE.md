# 王宗仁IP进化项目

## 项目背景

**律师**：王宗仁，北京市鼎业（南宁）律师事务所
**执业方向**：建设工程纠纷、税务合规、民事执行
**目标**：在抖音、微信视频号、小红书等平台打造律师个人IP
**定位**：不是"法律科普号"，而是"能解决真问题的专业律师"

**本项目**：通过系统学习安先生短视频方法论，拆解爆款文案，迭代分析模型，最终为律师IP定制文案策略。

---

## 总体逻辑

四步循环：**学 → 拆 → 化 → 用**

| 步骤 | 内容 | 产出 |
|------|------|------|
| 学 | 学习安先生课程文稿 | 课程总框架 + 每个课件的详细笔记 |
| 拆 | 用课程框架拆解爆款文案 | 每条文案的完整拆解记录 |
| 化 | 课程理论与拆解实战对照 | 迭代模型（课程理论的落地验证记录） |
| 用 | 为律师IP定制文案策略 | IP文案策略 + 写/改/仿写文案 |

## 各环节操作规则

### 学（新课件）

1. 用户发送课件文稿
2. 深度学习，提炼核心框架
3. 在 `02-知识库/课程学习/` 下新建笔记文件：`笔记XX-课件名.md`
4. 同步更新 `课程总框架.md`（精炼版，合并所有课件）

### 拆（爆款文案）

**两种方式**：

1. **单条拆解**：用户在对话中直接发送文案 + 说"拆解"
2. **批量拆解**：用户把文案文件放入 `01-待拆解/` 目录，说"拆解待处理"或"处理队列"

**批量拆解流程**：
- 读取 `01-待拆解/安先生/` 或 `01-待拆解/律师/` 目录下文件
- 用并行agent同时拆解多条（建议每次2-3条）
- 安先生文案写入 `02-知识库/拆解记录/安先生/`
- 律师文案写入 `02-知识库/拆解记录/律师/`
- 同步更新迭代模型
- 拆解完成后删除待拆解目录中的原文件

**拆解目的**：纯落地对照——课程方法论如何在文案中体现，不做任何延伸

**拆解固定板块**（只对照课程框架）：

1. **钩子落地对照**
   - 开篇叠加了哪些钩子？（对照课程13种钩子类型）
   - 全程钩子分布特征？（验证"钩子是成分不是环节")

2. **骨架落地对照**
   - 使用了哪种骨架？（对照课程6种骨架）
   - 是否叠加/切换？（验证"骨架可叠加"）

3. **势能意识落地对照**
   - 全篇核心概念是什么？
   - 每段是否为核心概念积累势能？
   - 有无削弱势能的废话？

4. **颗粒度意识落地对照**
   - AB是否分明？
   - 有无蔓延/过渡自然？
   - 数字/概念是否精确？

5. **首尾呼应落地对照**
   - 开头概念是什么？
   - 结尾如何回归？（验证"首尾闭环"）

6. **情绪刺点落地对照**
   - 点赞刺点在哪里？
   - 评论刺点在哪里？
   - 收藏刺点在哪里？
   - 转发刺点在哪里？
   - 关注刺点在哪里？

**不做**：
- 不总结"课程未覆盖的补充手法"
- 不归纳"谱系""技法"
- 不创造超出课程的新概念

**形式原则提炼**（在6个板块后增加）：

在拆解末尾增加"形式原则提炼"部分，只记录结构规律，不记录具体内容：

```
## 形式原则提炼（跨领域可迁移）

### 开篇模式
- 模式名称：xxx式开场
- 结构特征：（只写结构，不写具体内容）
  - 钩子组合类型：
  - 句式特征：
- 适用场景：xxx类内容

### 骨架推进模式
- 模式名称：xxx式推进
- 结构特征：（只写推进逻辑，不写具体链条）
  - 推进方式：
  - 节奏设计：
- 适用场景：xxx类内容

### 情绪刺点模式
- 模式名称：xxx式刺点预埋
- 结构特征：（只写预埋位置和触发方式）
  - 预埋位置：
  - 触发方式：
- 适用场景：xxx类内容
```

拆解完成后：
- 写入 `02-知识库/拆解记录/安先生/` 或 `02-知识库/拆解记录/律师/`
- **素材提取判断**（仅限律师文案）：
  - 是否包含可提取的案例？→ 追加到 `素材积累/[执业方向]/案例.md`
  - 是否包含可提取的痛点场景？→ 追加到 `素材积累/[执业方向]/痛点场景.md`
  - 是否包含可提取的行业潜规则？→ 追加到 `素材积累/[执业方向]/行业潜规则.md`
  - 是否包含可提取的常见误区？→ 追加到 `素材积累/[执业方向]/常见误区.md`
  - 原文存入 `素材积累/[执业方向]/素材原文/`
  - 更新索引
- 更新迭代模型中的"拆解验证记录"表格
- 更新迭代模型中的"形式原则库"（每拆解5条更新一次）

**两种拆解路径**：

1. **安先生文案拆解**：课程验证 + 形式原则提炼
   - 不记录具体内容，只记录结构规律
   - 目的：掌握课程方法论的"形式层"，可跨领域迁移

2. **律师文案拆解**：课程验证 + 可迁移模式 + 素材提取
   - 可记录具体内容（同领域）
   - 目的：掌握律师内容的"内容层"，直接可用
   - **额外步骤**：同步提取干货内容到素材积累

### 化（迭代模型）

每拆解5-10条文案后做一次**阶段性总结**：

- 哪些课程理论被反复验证
- 当前拆解进度

更新 `02-知识库/迭代模型.md`。

**不做**：不创造超出课程范围的新模型、新谱系、新技法。

### 用（写文案）

**约束确认**：动笔前先问清楚
- 目标用户是谁？什么状态？痛点是什么？
- 风格选择：导师式/诗人式/读书人式/修行者式/教师式
- 是否有素材要求（用哪个案例/法条）

**平台说明**：抖音、视频号、小红书使用**同一文案**，平台差异通过发布时的剪辑/封面/标题适配，文案本身不分裂。

**写作流程**（叙事线式）：

```
1. 确定核心概念 + AB张力
   - 核心概念是什么？（一句话可概括）
   - AB对立是什么？（A=常识/误区，B=作者立场）
   - 选择开篇模式（查迭代模型中的"叙事流畅性规律"）

2. 【第一次查素材】查框架型素材
   - 查"常见误区.md" → 选1条作为AB张力的A端
   - 查"痛点场景.md" → 选1条作为开篇场景（若选"痛点场景代入式"开篇）
   - **若无匹配素材**：问用户"没有找到匹配的误区/痛点场景素材，是否手动提供相关素材？"

3. 选择叙事推进模式
   - 逻辑+共鸣双轨推进？（理论→故事）
   - 螺旋上升式？（抽象→例证→方法→升维）
   - 场景枚举+节拍推进？（多个场景形成重复节拍）
   - 压抑积累→宣言释放？（前半段压抑→后半段释放）

4. 设计势能曲线
   - 开头：在哪积累势能？（否定/追问/批判）
   - 中段：否定链推进还是场景枚举？
   - 峰值：在哪达到势能峰值？（金句/古诗词/权威背书）
   - 结尾：在哪释放？（行动召唤/话说一半/升维）

5. 【第二次查素材】查填充型素材
   - 查"案例.md" → 选1-3条作为势能峰值案例
   - 查"行业潜规则.md" → 选2-3条形成否定链
   - 素材引用：根据文案需要精简或扩展
   - **若无匹配素材**：问用户"没有找到匹配的案例/行业潜规则素材，是否手动提供相关素材？"

6. 写初稿（用句子咬合技术）
   - 用否定词咬合："但是""然而""其实""并非...而是"
   - 零过渡跳跃：段落直接硬切，不用"首先其次最后"
   - 意象锚定+抽象提炼：先让观众看见，再让观众理解

7. 钩子和刺点顺势而化
   - 钩子嵌入叙事动线：否定性力量钩子通过案例承载，与我相关钩子通过"你"句式串入
   - 刺点预埋势能释放点：金句放势能峰值，话说一半放结尾
   - 收藏刺点：证据清单、维权路径、避坑指南等"以后用得上"的实用技巧

8. 按检验清单检查
```

**素材类型与用途对应**：

| 素材类型 | 查询时机 | 用途 | 查条数 | 引用方式 |

| 案例.md | 第5步 | 势能峰值、收藏刺点 | 1-3条 | 精简或扩展 |
| 行业潜规则.md | 第5步 | 否定链设计 | 2-3条 | 精简或扩展 |
| 法条.md | 按需 | 权威背书、论证依据 | 不作为刺点 | 精简或扩展 |

**自检清单**：

钩子检查：
- [ ] 开头2秒是否叠加了至少3种钩子？

### 用（改文案）

用户发送原稿 + 修改需求。

**改稿流程**：
1. 按拆解框架快速分析原稿，定位问题
2. 输出**改写后的完整文案**（不是标注修改点，而是给出完整版）
3. 附上简要说明：改了什么、为什么改

**常见问题定位**：
- 钩子不足：开头太平，中段无钩子
- 骨架松散：推进动力不足，信息密度低
- 缺少情绪刺点：全程平铺，没有触发互动的点
- 势能泄漏：中间插入了无关信息
- 与用户无关：讲产品/讲自己，不讲用户场景
- 颗粒度模糊：过度解释，分支太多

### 用（仿写文案）

用户发送原稿 + 说"仿写"或"改写"。

**仿写定位**：
- 与"改文案"不同：改文案是修改问题，仿写是换表达方式
- 核心内容不变，表达方式改变，避免抄袭

**仿写流程**：
1. 分析原稿结构：骨架类型、钩子分布、情绪刺点
2. 提取核心信息点：哪些必须保留
3. 重构表达：换一种说法、换一种骨架、换一种钩子组合
4. 输出仿写后的完整文案
5. 附上简要说明：保留了什么、改了什么

**仿写策略**（至少执行2-3项）：
- 换骨架：原稿用逻辑性推进，仿写用兴趣点推进或共鸣性推进
- 换钩子：原稿用否定性力量开场，仿写用反常或与我相关开场
- 换语气：原稿是导师式，仿写是读书人式或陪伴者式
- 换句式：原稿用排比，仿写用对偶或长短句交替
- 换案例：原稿用A案例说明，仿写用B案例说明同一观点
- 保留核心观点：原稿的核心论点必须保留，只是换个说法

**检验标准**：
- 核心观点是否一致？
- 表达方式是否明显不同？
- 是否避免了逐句抄袭？

### 素材积累

**核心设计**：存入时用户零负担，调用时按功能高效取用

**目录结构**：
```
素材积累/
├── 00-索引.md
├── 建工纠纷/
│   ├── 案例.md            ← 可直接引用的案情摘要
│   ├── 法条.md            ← 可直接引用的法条内容
│   ├── 行业潜规则.md      ← 可直接引用的潜规则提炼
│   ├── 常见误区.md        ← 可直接引用的误区提炼
│   ├── 痛点场景.md        ← 可直接引用的痛点提炼
│   └── 素材原文/          ← 完整原文存放备查
│       ├── 2026-04-26-001.md
│       └── ...
├── 税务合规/
│   └── （同上）
└── 民事执行/
    └── （同上）
```

**存入流程**：

用户发素材 + 说"整理素材" → Claude解析素材：
1. 判断执业方向（建工/税务/执行）
2. 判断包含哪些功能（案例/法条/潜规则/误区/痛点）
3. 同一素材拆分提取，分别存放对应功能文件
4. 原文完整存入素材原文目录（文件名：日期-编号）
5. 更新索引（关键词、统计数量）

用户无需判断分类，只管发素材。

**检测新素材流程**：

用户说"整理新素材"或"检测新素材" → Claude自动检测并处理：
1. 列出来源目录（如`/Users/Barry/Documents/多设备同步/knowledge/construction-law/raw/`）所有文件
2. 读取索引文件中的"素材原文清单"
3. 对比识别新增文件
4. 按存入流程处理新素材：解析 → 拆分提取 → 追加存入功能文件 → 原文备份 → 更新索引
5. 报告处理结果：新增多少篇、提取多少条

**调用流程**：

写文案时 → 判断执业方向 → 根据需要的功能直接读取对应文件 → 关键词匹配 → 取用"核心内容"字段直接引用

```markdown

【来源】2026-04-26-001（对应素材原文文件名）
【一句话描述】（50字以内）
【核心内容】（100-300字，可直接引用）
```

**素材原文格式**：

```markdown
【来源类型】随手记/专业文章/短视频文案/判例案例
【存入时间】2026-04-26
```
