# 王宗仁IP进化项目

## 项目背景

**律师**：王宗仁，北京市鼎业（南宁）律师事务所
**执业方向**：建设工程纠纷、税务合规、民事执行
**目标**：在抖音、微信视频号、小红书等平台打造律师个人IP
**定位**：不是"法律科普号"，而是"能解决真问题的专业律师"

**本项目**：通过系统学习安先生短视频方法论，拆解爆款文案，迭代分析模型，最终为律师IP定制文案策略。

---

## 总体逻辑

四步循环：**学 → 拆 → 化 → 用**

| 步骤 | 内容 | 产出 |
|------|------|------|
| 学 | 学习安先生课程文稿 | 课程总框架 + 每个课件的详细笔记 |
| 拆 | 用课程框架拆解爆款文案 | 每条文案的完整拆解记录 |
| 化 | 课程理论与拆解实战融合 | 迭代模型（已验证的理论 + 补充手法 + 共性特征） |
| 用 | 为律师IP定制文案策略 | IP文案策略 + 写/改/仿写文案 |

## 各环节操作规则

### 学（新课件）

1. 用户发送课件文稿
2. 深度学习，提炼核心框架
3. 在 `02-知识库/课程学习/` 下新建笔记文件：`笔记XX-课件名.md`
4. 同步更新 `课程总框架.md`（精炼版，合并所有课件）

### 拆（爆款文案）

**两种方式**：

1. **单条拆解**：用户在对话中直接发送文案 + 说"拆解"
2. **批量拆解**：用户把文案文件放入 `01-待拆解/` 目录，说"拆解待处理"或"处理队列"

**批量拆解流程**：
- 读取 `01-待拆解/` 目录下所有文件
- 用并行agent同时拆解多条（建议每次2-3条，避免网络中断导致全部丢失）
- 结果写入 `02-知识库/拆解记录/`
- 同步更新迭代模型和IP策略
- 拆解完成后删除待拆解目录中的原文件

**拆解固定板块**：

1. **钩子分析（前2秒）**：开篇叠加了哪些钩子类型
2. **全文钩子分布**：逐段标注钩子位置（课程核心认知：钩子是成分不是环节）
3. **骨架分析**：主骨架类型 + 叠加骨架类型
4. **情绪刺点预埋清单**：点赞/评论/收藏/转发/关注刺点
5. **骨架三大意识体现**：观众意识、颗粒度意识、势能意识
6. **与之前拆解对比**：找出差异与共性
7. **课程未覆盖的补充手法**：实战中发现的新技法
8. **对律师IP的迁移价值**：可直接用于律师内容的方法

拆解完成后：
- 写入 `02-知识库/拆解记录/拆解XXX-xxx.md`
- 同步更新 `02-知识库/迭代模型.md`
- 同步更新 `02-知识库/IP文案策略.md`

### 化（迭代模型）

每拆解5-10条文案后做一次**阶段性总结**：

- 哪些课程理论被反复验证
- 哪些理论需要修正或补充
- 当前分析模型是否需要升级
- 对IP文案策略的建议是否需要调整

更新 `02-知识库/迭代模型.md`。

### 用（写文案）

**约束确认**：动笔前先问清楚
- 目标用户是谁？什么状态？痛点是什么？
- 风格选择：导师式/诗人式/读书人式/修行者式/教师式
- 是否有素材要求（用哪个案例/法条）

**平台说明**：抖音、视频号、小红书使用**同一文案**，平台差异通过发布时的剪辑/封面/标题适配，文案本身不分裂。

**写作流程**：
1. 翻阅 `02-知识库/迭代模型.md` 和 `02-知识库/IP文案策略.md` 选择骨架和钩子
2. 翻阅 `02-知识库/素材积累/[执业方向]/` 下对应功能文件查找可用素材
3. 写初稿，标注每个位置用了什么钩子/骨架/刺点
4. 按自检清单检查

**自检清单**：

钩子检查：
- [ ] 开头2秒是否叠加了至少3种钩子？
- [ ] 中段是否持续有钩子？
- [ ] 钩子类型是否匹配目标用户？

骨架检查：
- [ ] 是否有明确的主骨架推进？
- [ ] 是否叠加了第二种骨架？
- [ ] 是否有势能积累→关键场面释放？

情绪刺点检查：
- [ ] 是否预埋了点赞刺点？
- [ ] 是否预埋了评论刺点（站队/话说一半/提问）？
- [ ] 是否预埋了收藏刺点？
- [ ] 是否预埋了关注刺点？

三大意识检查：
- [ ] 观众意识：是否有喘息点？
- [ ] 颗粒度意识：AB分明，没有黏糊糊的过渡？
- [ ] 势能意识：有没有削弱势能的废话？

### 用（改文案）

用户发送原稿 + 修改需求。

**改稿流程**：
1. 按拆解框架快速分析原稿，定位问题
2. 输出**改写后的完整文案**（不是标注修改点，而是给出完整版）
3. 附上简要说明：改了什么、为什么改

**常见问题定位**：
- 钩子不足：开头太平，中段无钩子
- 骨架松散：推进动力不足，信息密度低
- 缺少情绪刺点：全程平铺，没有触发互动的点
- 势能泄漏：中间插入了无关信息
- 与用户无关：讲产品/讲自己，不讲用户场景
- 颗粒度模糊：过度解释，分支太多

### 用（仿写文案）

用户发送原稿 + 说"仿写"或"改写"。

**仿写定位**：
- 与"改文案"不同：改文案是修改问题，仿写是换表达方式
- 核心内容不变，表达方式改变，避免抄袭

**仿写流程**：
1. 分析原稿结构：骨架类型、钩子分布、情绪刺点
2. 提取核心信息点：哪些必须保留
3. 重构表达：换一种说法、换一种骨架、换一种钩子组合
4. 输出仿写后的完整文案
5. 附上简要说明：保留了什么、改了什么

**仿写策略**（至少执行2-3项）：
- 换骨架：原稿用逻辑性推进，仿写用兴趣点推进或共鸣性推进
- 换钩子：原稿用否定性力量开场，仿写用反常或与我相关开场
- 换语气：原稿是导师式，仿写是读书人式或陪伴者式
- 换句式：原稿用排比，仿写用对偶或长短句交替
- 换案例：原稿用A案例说明，仿写用B案例说明同一观点
- 保留核心观点：原稿的核心论点必须保留，只是换个说法

**检验标准**：
- 核心观点是否一致？
- 表达方式是否明显不同？
- 是否避免了逐句抄袭？

### 素材积累

**核心设计**：存入时用户零负担，调用时按功能高效取用

**目录结构**：
```
素材积累/
├── 00-索引.md
├── 建工纠纷/
│   ├── 案例.md            ← 可直接引用的案情摘要
│   ├── 法条.md            ← 可直接引用的法条内容
│   ├── 行业潜规则.md      ← 可直接引用的潜规则提炼
│   ├── 常见误区.md        ← 可直接引用的误区提炼
│   ├── 痛点场景.md        ← 可直接引用的痛点提炼
│   └── 素材原文/          ← 完整原文存放备查
│       ├── 2026-04-26-001.md
│       └── ...
├── 税务合规/
│   └── （同上）
└── 民事执行/
    └── （同上）
```

**存入流程**：

用户发素材 + 说"整理素材" → Claude解析素材：
1. 判断执业方向（建工/税务/执行）
2. 判断包含哪些功能（案例/法条/潜规则/误区/痛点）
3. 同一素材拆分提取，分别存放对应功能文件
4. 原文完整存入素材原文目录（文件名：日期-编号）
5. 更新索引（关键词、统计数量）

用户无需判断分类，只管发素材。

**检测新素材流程**：

用户说"整理新素材"或"检测新素材" → Claude自动检测并处理：
1. 列出来源目录（如`/Users/Barry/Documents/多设备同步/knowledge/construction-law/raw/`）所有文件
2. 读取索引文件中的"素材原文清单"
3. 对比识别新增文件
4. 按存入流程处理新素材：解析 → 拆分提取 → 追加存入功能文件 → 原文备份 → 更新索引
5. 报告处理结果：新增多少篇、提取多少条

**调用流程**：

写文案时 → 判断执业方向 → 根据需要的功能直接读取对应文件 → 关键词匹配 → 取用"核心内容"字段直接引用

**功能文件格式**：

```markdown
【关键词】xxx xxx
【来源】2026-04-26-001（对应素材原文文件名）
【一句话描述】（50字以内）
【核心内容】（100-300字，可直接引用）
```

**素材原文格式**：

```markdown
【来源类型】随手记/专业文章/短视频文案/判例案例
【存入时间】2026-04-26
【原文】（完整内容）
```

**示例**：一条素材的拆分存放

用户发：
> 南宁工程老板被总包拆分人材机拒付工程款，第一次开庭证据不足被问住，后来委托我们重新组织证据

Claude拆分存放：
- 案例.md：案情摘要
- 行业潜规则.md：拆分人材机规避违法转包的套路
- 痛点场景.md：明明干了活却被推出门外的困境
- 素材原文/：完整原文备查
