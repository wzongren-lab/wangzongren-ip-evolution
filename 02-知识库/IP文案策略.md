# 律师IP文案策略

> 初始版本：2026-04-24
> 律师：王宗仁，北京市鼎业（南宁）律师事务所
> 执业方向：建设工程纠纷、税务合规、民事执行

---

## IP定位

不是"法律科普号"，而是"能解决真问题的专业律师"。

---

## 平台策略

抖音、视频号、小红书使用**同一文案**。平台差异通过发布时的剪辑节奏/封面风格/标题适配，文案本身不分裂。

文案的核心质量（钩子、骨架、情绪刺点）是通用的，不需要为不同平台写不同版本。

---

## 执业方向的文案切入点

### 建设工程纠纷
- 痛点明确：甲方拖欠工程款、结算争议、工期延误索赔
- 受众：施工企业老板、项目经理、分包商
- 可用钩子：否定性力量（"你签了这个条款，活该拿不到钱"）、与我相关（点名人群）

### 税务合规
- 痛点明确：税务稽查风险、发票合规、税负优化
- 受众：企业老板、财务负责人
- 可用钩子：否定性力量（"你这样做账，等税务局来查就晚了"）、窥探（行业内幕）

### 民事执行
- 痛点明确：赢了官司拿不到钱、被执行人转移财产
- 受众：债权人、胜诉当事人
- 可用钩子：否定性力量（"赢了官司不等于拿到钱"）、与我相关（"打官司赢了的人注意"）

| 021 | 场景重复变奏——法律困境螺旋推进 | 用3-4个不同当事人案例呈现同一法律困境（签合同时/纠纷时/执行时），每次略有变化，最后升到普遍困境 |
| 021 | 概念正名进阶——给负面行为赋予正面意义 | "表达就是筛选灵魂"→律师版："你不是'怕输'，你是'谨慎'""你不是'计较'，你是'在保护自己'" |

| 021 | 直接感谢作为关注刺点 | "谢谢你看到这里，说明你在认真对待自己的权利"——感谢+身份认同 |

| 022 | 问句开篇+故事跟进 | "连续三年处理建设工程纠纷是一种怎样的体验？"——问句制造好奇，立刻进入律师故事 |
| 022 | 自我经历作为权威替代 | 律师不需要每次都引用法条，可以用"我办过XX个案子"作为经验权威——"我就是证据" |
| 022 | 身体感知细节——当事人共鸣 | "在法院门口等开庭，那种胃里空荡荡的感觉"——调用当事人的身体记忆，制造共鸣 |
| 022 | 时间序列势能——律师经历 | "这个案子从立案到执行，我们走了三年"——时间序列本身就制造分量 |
| 022 | 名言完整引用——仪式感收束 | 引用法条/判例原文完整引用，制造仪式感和可截图性 |
| 023 | 配对式论证——法律概念对比 | "你以为这是西方的法律概念？其实我们的民法典早就有相似规定"——日常逻辑→法律概念配对 |
| 023 | 原文美学功能——法条朗读力量 | 法条原文一字不改地朗读——让法律语言本身成为美感触发点 |
| 023 | 七层共鸣阶梯——法律内容深度 | 从当事人最直接的痛点（钱/时间）→一路钻到存在层面（你想要什么样的生活） |
| 023 | "如果你X，请记住Y"公式 | "如果你担心对方转移财产，请记住《民事诉讼法》第100条……"——实用指引标准公式 |
| 023 | 争议性转折——法律热点辨析 | "网上说'打官司一定要请律师'，其实……"——用争议点制造转折 |
| 024 | "你→我"主语切换——律师版 | 前半段描述当事人的困境（"你签合同时没注意这条……"），后半段用律师视角给出解决方案（"所以我帮你……"） |
| 024 | 三连场景积累势能——当事人困境 | 三个当事人常见困境场景（签合同/打官司/执行），形成"法律风险无处不在"的压迫感 |

| 025 | 数字预告结构——律师案例枚举 | "这一年，我办了七个让我睡不着的案子"——数字预告+清单式枚举，完播驱动+专业信任 |

| 025 | 自我暴露建立信任——律师脆弱时刻 | "我独自整理卷宗到凌晨三点，但我羡慕那些当事人，因为他们至少还有人值得打官司"——承认脆弱建立信任 |
| 025 | 场景间无过渡直接跳跃 | 多个当事人故事之间不用"还有一个案子"过渡，直接用"第三个是……"硬切 |

| 学者式 | 疑问开场、配对论证、原文美学、七层共鸣阶梯 | 文化对比、知识溯源、深度论证内容 | 023（中西哲学对比） |

**律师IP建议**：多种风格交替使用。

- **导师式**：硬核法律干货、风险警示、实操指南——建立专业可信度（攻击性强，建立权威）
- **批判者式**：热点辨析、法律纠偏、诊断式分析——建立独到见解（批判热点，给出正解，适合"网上炒的法律概念可能是错的"类内容）
- **鉴赏家式**：法律溯源、法条谱系、跨学科映射——建立知识广度（带着观众"去看"，适合"法条从哪来""判例的基因"类内容）
- **诗人式**：正义观、职业信念、对当事人的理解——建立精神价值（温情脉脉，建立信任）
- **读书人式**：典型案例故事、法条背后的人性、当事人的处境——建立情感连接（共鸣优先）
- **修行者式**：法律思维的自我觉察、从"不懂法"到"有法律意识"的成长——建立共鸣和转发（治愈导向）
- **教师式**：系统教学方法论、从原理到实操的完整拆解——建立专业深度（干货优先）
- **美学家式**：法律概念深度解析、颠覆常识认知、法理审美——建立思想高度（论证优先，适合建设工程纠纷的复杂法律论证）
- **布道者式**：宣告法律的意义、不引用法条的纯信念内容——建立人设锚点（信念优先，适合"为什么成为律师""法律的意义是什么"类内容）
- **陪伴者式**：牵手当事人走过法律之路、减轻心理负担——建立情感信任（温柔陪伴，适合"你的计较是护""你的谨慎是智慧"类内容）
- **哲学家式**：批判社会趋势、给负面状态正名、从现象到存在的深度论证——建立思想深度（批判趋势、价值观召唤，适合"为什么'计较'是在护""为什么法律不信任'口头承诺'"类内容）
- **夜游者式**：展示律师的孤独时刻、不教不宣只展示——建立人格魅力（喃喃自语，适合"凌晨看案卷""开庭前的沉默"类内容，展示"律师也是人"）

不过度依赖单一风格，保持账号内容多样性。

### 内容谱系：从硬到软

十条拆解揭示了一个完整的语气谱系——从攻击性最强到最温柔：

```
硬 ←————————————————————————————————————————————→ 软

批判者  导师式  鉴赏家  教师式  美学家  诗人式  读书人  修行者  布道者  陪伴者  哲学家  夜游者
 009    001    008    005    006    002    003    004    007    010    016    018
```

- **硬端**：批判、否定、纠正——建立权威和独到见解
- **软端**：陪伴、召唤、接纳——建立信任和情感连接
- **一个账号需要覆盖整个谱系**——全是硬的会吓跑人，全是软的会显得没干货

### 内容功能谱系

| 功能 | 代表拆解 | 观众获得 | 账号价值 | 律师应用 |
|------|----------|----------|----------|----------|
| 教方法 | 001/005/008 | "我会用了" | 工具价值→关注 | 教法律操作方法 |
| 传认知 | 006/009 | "我理解了" | 思想价值→收藏 | 传法律底层逻辑 |
| 递情感 | 002/003/004 | "我被触动了" | 情感价值→转发 | 递当事人故事 |
| 宣信念+陪伴 | 007/010 | "我认同了" | 精神价值→深度关注 | 宣法律信念+陪伴当事人 |

**律师账号的分布建议**：
- 50%教方法+传认知（导师式+批判者式+鉴赏家式+教师式+美学家式）——建立专业能力可信度
- 25%递情感+陪伴（读书人式+修行者式+陪伴者式）——建立情感连接和信任
- 15%信念端（布道者式+诗人式）——建立人设锚点
- 10%热点端（批判者式为主）——蹭热点并正名，获取流量

**信念端内容不需要多，但必须有**——007在安先生账号里只占一条，但它是人设锚点，让观众从"学习这个账号"跨越到"认同这个账号"。
